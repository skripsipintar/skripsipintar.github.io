3. jelaskan query yang termasuk dalam DDL


DDL adalah kependekan dari Data Definition Language. Query yang termasuk dalam DDL adalah perintah - perintah yang berhubungan dengan skema struktur sebuah database.

Pembahasan

Beberapa contoh query yang termasuk pada Data Definition Language beserta penjelasannya antara lain sebagai berikut :

Create

perintah yang digunakan untuk membuat database dan juga isi di dalamnya seperti tabel atau view. Syntax atau format penulisan query create adalah sebagai berikut

CREATE DATABASE nama_database; CREATE TABLE nama_tabel  (nama_kolom1, type_kolom1,  nama_kolom2, type_kolom2,…);

Alter

perintah yang digunakan untuk melakukan perubahan pada tabel yang telah dibuat. Perubahan tersebut seperti menambah kolom, mengubah kolom, menghapus kolom, atau menambahkan primary dan foreign key. Format penulisan query tersebut adalah seperti berikut

ALTER TABLE nama_tabel  ADD nama_kolom tipe_data;ALTER TABLE nama_tabelDROP COLUMN nama_kolom;ALTER TABLE nama_tabel MODIFY COLUMN nama_kolom tipe_dataALTER TABLE nama_tabel  ADD PRIMARY KEY (nama_kolom);\

Drop

perintah yang digunakan untuk menghapus objek yang ada di dalam database seperti database, tabel, atau kolom. Contoh penggunaan perintah ini adalah

DROP DATABASE nama_database;DROP TABLE nama_table;

Truncate

perintah yang digunakan untuk mengosongkan atau menghapus semua records atau data yang ada pada sebuah tabel. Contoh penulisan perintah tersebut adalah sebagai berikut

TRUNCATE TABLE nama_tabel;

• Jawaban pendek:

Berikan contoh untuk basis data DDL dan DML!

DDL:

1. CREATE
Perintah ini digunakan untuk membuat, termasuk di antaranya membuat database baru, tabel baru, view baru, dan kolom.
Contoh : create table mahasiswa (nim  char(8) primary key,nama_mahasiswa  varchar(20),nilai integer(3),alamat  varchar(25);
2. ALTER
Perintah ini digunakan untuk mengubah struktur tabel yang telah dibuat. Pekerjaannya mencakup mengganti nama tabel, menambah kolom, mengubah kolom, menghapus kolom, maupun memberikan atribut pada kolom.
Contoh : alter table mahasiswa rename wisudawan;
3. DROP 
Perintah ini digunakan untuk menghapus database dan tabel.
Contoh : Alter table ‘mahasiswa’ drop ‘alamat’;

DML:

1. INSERT
Perintah ini digunakan untuk menyisipkan atau memasukkan data baru ke dalam tabel. Penggunaannya setelah database dan tabel selesai dibuat.Contoh : Insert into mahasiswa values (“08052926”, “Frenky”,”70”);

2. SELECT
Perintah ini digunakan untuk mengambil data atau menampilkan data dari satu tabel atau beberapa tabel dalam relasi. Data yang diambil dapat kita tampilkan dalam layar prompt MySQL secara langsung maupun ditampilkan pada tampilan aplikasi.Contoh : Select nama_mahasiswa from mahasiswa where nilai = 70;

3. UPDATE 
Perintah ini digunakan untuk memperbarui data lama menjadi data terkini. Jika Anda memiliki data yang salah atau kurang up to date dengan kondisi sekarang, maka dapat diubah isi datanya menggunakan perintah UPDATE.Contoh : mahasiswa set nim = ‘08052926’ = 08052927;

4. DELETE
Perintah ini digunakan untuk menghapus data dari tabel. Biasanya data yang dihapus merupakan data yang sudah tidak diperlukan lagi. Pada saat menghapus data, perintah yang telah dijalankan tidak dapat digagalkan, sehingga data yang telah hilang tidak dapat dikembalikan lagiContoh : delete form mahasiswa;

14. Apa itu DDL, DML, DCL dalam Basisdata dan jelaskan mengenai isinya?


Data Definition Language (DDL)

DDL adalah sub perintah dari bahasa SQL yang digunakan untuk membangun kerangka sebuah database, dalam hal ini database dan table. Terdapat tiga perintah penting dalam DDL, yaitu CREATE, ALTER, DROP.

CREATE: perintah ini digunakan untuk membuat, termasuk di dalamnya membuat database baru, tabel baru view baru, dan kolom baru.
Contoh: CREATE DATABASE nama_database

ALTER: perintah ALTER berfungsi untuk mengubah struktur tabel yang telah dibuat. Mencakup di dalamnya mengubah nama tabel, menambah kolom, mengubah kolom, menghapus kolom, dan memberikan atribut pada kolom.
Contoh: ALTER TABLE nama_tabel ADD nama_kolom datatype

DROP: perintah DROP berfungsi untuk menghapus database atau tabel.
Contoh: DROP DATABASE nama_database

Data Manipulation Language (DML)

DML adalah sub perintah dari bahasa SQL yang digunakan untuk memanipulasi data dalam database yang telah dibuat. Terdapat empat perintah penting dalam DML, yaitu INSERT, SELECT, UPDATE, dan DELETE.

INSERT: perintah ini digunakan untuk memasukkan data baru ke dalam sebuah tabel. Perintah ini tentu saja bisa dijalankan ketika database dan tabel sudah dibuat.
Contoh: INSERT INTO nama_tabel VALUES (data1, data2, dst…);

SELECT: perintah ini digunakan untuk mengambil dan menampilkan data dari tabel atau bahkan dari beberapa tabel dengan penggunaan relasi.
Contoh: SELECT nama_kolom1, nama_kolom2 FROM nama_tabel;

UPDATE: perintah update digunakan untuk memperbaharui data pada sebuah tabel.
Contoh: UPDATE nama_tabel SET kolom1=data1, kolom2=data2,… WHERE kolom=data;

DELETE: perintah delete digunakan untuk menghapus data dari sebuah tabel.
Contoh: DELETE FROM nama_tabel WHERE kolom=data;

Data Control Language (DCL)

DCL adalah sub bahasa SQL yang berfungsi untuk melakukan pengontrolan data dan server databasenya, seperti manipulasi user dan hak akses (priviledges). Yang termasuk perintah dalam DCL ada dua, yaitu GRANT dan REVOKE.

GRANT: perintah ini digunakan untuk memberikan hak akses oleh admin ke salah satu user atau pengguna. Hak akses tersebut bisa berupa hak membuat (CREATE), mengambil data (SELECT), menghapus data (DELETE), mengubah data (UPDATE), dan hak khusus lainnya yang berhubungan dengan sistem database.

REVOKE: perintah ini digunakan untuk mencabut hak akses yang telah diberikan kepada user. Dalam ini merupakan kebalikan dari perintah GRANT.

18. perbedaan ddl,dml,dcl,tcl


Adapun perbedaan dari DDL, DML, DCL, dan TCL itu terletak pada pengertian dari masing sejenis nya sendiri, Yang mana dapat kita jelaskan sebagai berikut:

Data Definition Language atau sering di singkat juga dengan DDL dapat kita artikan bahwa DDL dapat digunakan supaya dapat mengubah dan juga memodifikasi suatu struktur database dan juga sebuah tabel dan skema. sehingga dari hal tersebut dapat mengatasi permasalahan di bidang desain dan juga penyimpanan objek database.Bahasa Manipulasi Data  atau sering di singkat juga dengan DML dapat kita artikan sebagai suatu ajuan untuk sebuah perintah yang terdapat di dalam suatu bahasa pemrograman SQL yang mana hal tersebut digunakan untuk dapat memanipulasi data dalam sebuah database.Data Control Language atau sering di singkat juga dengan DCL dapat kita artikan sebagai suatu ajuan untuk sebuah perintah yang terdapat di dalam suatu bahasa pemrograman SQL yang mana hal tersebut digunakan untuk dapat merubah hak akses, memberikan sebuah roles, dan juga mengenai sebuah isu lain yang berhubungan dengan keamanan database sehingga hal tersebut dapat berguna untuk pengontrolam dan memanipulasi dari hak akses database itu sendiri.Transaction Control Language atau sering di singkat juga dengan TCL ialah Perintah TCL berurusan dengan transaksi dalam database. Melakukan sebuah query untuk transaksi dalam sebuah logika database.

24. apa yang dimaksud dengan query?


Query adalah suatu bentuk pemanggilan data yang ada pada database. Pemanggilan data menggunakan query harus memenuhi format dan aturan yang sesuai dengan ketentuan Structure Query Language/SQL agar database dapat membaca pemanggilan secara tepat. Penggunaan query dapat digunakan untuk melakukan menambahkan data, mengupdate data dan melakukan penghapusan data.

Pembahasan

Query merupakan suatu hal yang identik dengan database karena keduanya memiliki hubungan. Tanpa query, data yang tersimpan dalam database tidak dapat dilakukan manipulasi terhadap isi didalamnya karena diibaratkan pengguna melakukan requesting terhadap database untuk melakukan tindakan terhadap data didalamnya. Contoh query adalah penggunaan SQL statement seperti SELECT [column, ...] FROM [table name]. Untuk mengakses data dalam database, ada cara lain selain menggunakan query, namun, query tetap dianggap solusi paling efektif dalam melakukan pemanggilan data di database.

1. Keterkaitan antar komponen dari Sistem Basis Data sangat penting karena setiap komponen memiliki peran dan tanggung jawabnya masing-masing dalam memastikan basis data berjalan dengan baik. Misalnya, struktur tabel (table structure) yang terdiri dari kolom (column) dan tipe data (data type) harus diatur dengan benar agar data bisa disimpan dengan tepat dan valid. Selain itu, keterkaitan antara bahasa query (query language) dengan model data (data model) harus dijaga agar data bisa diambil dan dimanipulasi dengan benar. Jika salah satu komponen tidak berfungsi dengan benar, bisa berdampak pada kinerja dan integritas basis data secara keseluruhan.

3. Berikut adalah penjelasan singkat tentang beberapa istilah dalam basis data:

• Bit: satuan terkecil dalam komputasi yang bernilai 0 atau 1.

• Byte: kumpulan 8 bit yang mewakili sebuah karakter atau simbol.

• Field: bagian terkecil dalam sebuah tabel yang menyimpan informasi tentang suatu aspek atau atribut dari entitas atau objek yang diwakilinya.

• Record: kumpulan data atau informasi dalam sebuah tabel yang mewakili satu entitas atau objek.

• File: kumpulan data yang diorganisir secara logis dan memiliki nama serta lokasi tertentu pada sistem operasi.

• Basis data (database): kumpulan data atau informasi yang disimpan dalam sistem komputer dengan tujuan untuk diakses, dimanipulasi, dan dikelola.

• Sistem basis data (database system): kumpulan perangkat lunak dan perangkat keras yang digunakan untuk mengelola dan menyimpan data dalam basis data.

4. DDL (Data Definition Language) adalah bahasa basis data yang digunakan untuk mendefinisikan struktur dan schema dalam basis data. Contoh DDL adalah:

• CREATE TABLE: membuat tabel baru dalam basis data

• ALTER TABLE: mengubah struktur tabel, misalnya menambahkan kolom atau mengubah tipe data

• DROP TABLE: menghapus tabel dari basis data

DML (Data Manipulation Language) adalah bahasa basis data yang digunakan untuk mengambil, memperbarui, dan menghapus data dalam basis data. Contoh DML adalah:

• SELECT: mengambil data dari satu atau beberapa tabel

• INSERT INTO: menambahkan data ke dalam tabel

• UPDATE: memperbarui data dalam tabel

• DELETE: menghapus data dari tabel.

Apa Perbedaan Antara DDL dan DML?

Ringkasan -DDL vs DML

Database Relasional adalah jenis basis data yang umum. Structured Query Language (SQL) digunakan untuk melakukan operasi seperti menyimpan, memanipulasi dan mengambil data dari database relasional. SQL memiliki tiga subkategori utama. Mereka adalah DDL, DML dan DCL. Perbedaan antara DDL dan DML adalah bahwa DDL digunakan untuk mengubah struktur database dan DML digunakan untuk mengelola data dalam database.
